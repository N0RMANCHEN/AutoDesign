# AutoDesign Product Standards

> 作用：定义产品定位、默认行为、交付优先级和设计质量要求。  
> 定位：长期产品规则；不替代 `Roadmap` 的执行排期，也不替代 `Test-Standards` 的验证细则。

## 1. 产品定位

`AutoDesign` 解决两类问题：

1. 让 AI 通过本地插件和 bridge 安全修改 Figma
2. 让 Figma 里的设计事实更稳定地转成前端改造输入

它当前不是：

- 完整设计平台
- 多人协作 SaaS
- 无需审阅的全自动设计生成器

## 2. 非协商产品原则

### 2.1 可编辑优先

只要任务目标是 UI 还原，正式交付优先为可编辑节点：

- `FRAME`
- `GROUP`
- `TEXT`
- `RECTANGLE`
- `ELLIPSE`
- `LINE`
- `SVG`
- `COMPONENT / INSTANCE`

`raster` 只能作为诊断、辅助分析或过渡路径，不应冒充正式结果。

### 2.2 结构优先于贴图

对屏幕内 UI 回归，先保证：

- 容器结构
- 层级关系
- 间距和尺寸
- 设计 token
- 文本和规则图元

之后才考虑纹理、补图和高保真细节。

### 2.3 单父级小步修改

任何复杂 UI 修改或 reconstruction refine，默认一轮只收一个局部：

- 一个父级
- 一组容器
- 一组文本
- 一组规则图元

禁止跨多个父级盲目混改。

### 2.4 先观察再写入

执行 Figma 写操作前，默认先做：

1. 看当前 selection 或目标 Frame
2. 导出 preview
3. 明确目标节点和范围
4. 再写入

没有观察步骤的写入默认视为高风险。

### 2.5 清旧层优先于继续叠层

对 reconstruction 和重建设计类任务：

- 每轮 apply 前先清目标 Frame 内历史 AD 生成层
- 不允许把新一轮结果叠在旧错误层上
- 用户原始节点和 AD 生成节点必须可区分

### 2.6 定向写入优先

所有修改类外部命令都应默认面向指定节点：

- 明确 `nodeIds`
- 明确父级
- 明确目标 Frame

不允许把“当前 selection 全改了”当成默认安全行为。

## 3. Figma 执行面规则

### 3.1 安全边界

- 写操作必须经过插件 runtime 或正式受控的 capability 路径
- bridge 和 CLI 负责编排，不直接拥有 Figma 节点真相
- 所有修改都应有结构化结果或错误说明

### 3.2 插件 UI 规则

- 正式插件 UI 默认冻结
- 非用户明确要求，不修改可见布局、尺寸、文案
- UI 修改必须同步更新 UI lock 和相关文档

### 3.3 selection / inspect 规则

- 说“当前选中的图/对象/Frame”时，默认优先读取 Figma selection 或 subtree
- 不能把聊天附件和 Figma 当前 selection 混为一谈
- inspect 能力必须支持父子关系、布局属性、裁切、mask、组件信息

## 4. Reconstruction 交付优先级

对“目标 Frame + 参考图”的回归任务，正式优先级固定为：

1. 屏幕内 UI，非样机区域优先
2. 语义结构正确
3. 可编辑结果正确
4. 布局、尺寸、位置、颜色、圆角收敛
5. 文本和规则图元收敛
6. 未展示区域做同风格保守补全
7. 栅格诊断只作为辅助，不作为最终交付

## 5. 设计质量要求

### 5.1 设计 token 一致性

任何生成或重建结果都要向统一 token 收敛：

- 颜色阶
- 字号和字重层级
- 间距体系
- 圆角体系
- 透明度和阴影体系

### 5.2 大厂设计规范约束

默认以成熟移动端界面规范为基线：

- 间距有节奏
- 层级清晰
- 触达尺寸合理
- 对齐稳定
- 不出现随机视觉参数

### 5.3 推断内容的边界

参考图未展示区域允许补全，但必须：

- 风格一致
- 保守猜测
- 不引入与现有信息冲突的新功能块
- 标记为 `inferred`

## 6. 当前不允许的伪完成

以下结果不能视为完成：

- 只把图片变形后贴进 Frame，却声称“已矢量化”
- 结果层叠在旧错误层上
- 视觉差异明显，却只拿局部参数说明“差不多”
- 文本、层级、布局明显错误，却只靠全局平均分过关
